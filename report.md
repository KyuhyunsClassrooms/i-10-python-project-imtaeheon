# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제: 등차,등비수열의 합 계산기

이 프로그램을 만든 이유: 수학에 관심이 매우많고 복잡한 등차 또는 등비수열의 합을 구하고싶을때 또는 계산이 귀찮을때 사용한다


---

## 2. 문제 분해 및 모델링

### 입력
등차수열, 등비수열, 이전내역보기    
중에서 선택후 첫째항, 공차(혹은 공비), 계산 시작 항, 끝항

### 처리
등비수열 등차수열 각각 함수를 만들어놓고 (if문으로) 입력받은뒤 사용하는 방식
계산할때마다 2차원리스트에 내역을 추가한다

### 출력
합 출력.
또 다시 반복(종료되지않음)

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름: history

한 행의 의미: 사용자가 메뉴를 통해 단 한 번 수행한 '하나의 수열 계산 기록 세트

각 열의 의미:

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 | 등차 or 등비  |
| 1번 열 |  a(첫재항)  |
| 2번 열 | d or r (공차혹은 공비)   |
| 3번 열 |  n (시작)  |
ㅣ 4번열 ㅣ m(끝) ㅣ
ㅣ 5번열 ㅣ 결과 ㅣ

 이 2차원 리스트를 사용한 이유: 이전내역을 보고싶을때 꺼내서 볼수 있기때문

## 4. 함수 설명

첫 번째 함수: calculate_arithmetic_range

역할: 첫째항, 공차, 시작항, 끝항을 매개변수로 받아 등차수열의 특정 지정 구간의 합을 반복문으로 계산하여 반환합니다.

입력값: a (첫째항), d (공차), n (시작 항 번호), m (끝 항 번호)

반환값 또는 출력: 루프를 돌며 조건에 맞는 구간의 항들만 누적하여 더한 최종 등차수열 구간 합 결과값(total)을 반환(return)합니다.

2. 두 번째 함수: calculate_geometric_range

역할: 첫째항, 공비, 시작항, 끝항을 매개변수로 받아 등비수열의 특정 지정 구간의 합을 반복문으로 계산하여 반환합니다.

입력값: a (첫째항), r (공비), n (시작 항 번호), m (끝 항 번호)

반환값 또는 출력: 루프를 돌며 조건에 맞는 구간의 항들만 누적하여 곱하고 더한 최종 등비수열 구간 합 결과값(total)을 반환(return)합니다.

3. 세 번째 함수: show_history

역할: 수집된 2차원 리스트를 전달받아 그 내부에 쌓인 모든 데이터 행들을 순서대로 읽어와 화면에 예쁜 표 형태로 일괄 출력합니다.

입력값: history_list (데이터가 누적되어 온 2차원 리스트 history)

반환값 또는 출력: 별도의 반환값은 없으며, 리스트가 비었을 때는 안내 메시지를, 데이터가 있을 때는 인덱스(info[0] ~ info[5])를 활용해 가독성 높은 문장 양식으로 화면에 직접 출력(print)합니다.

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름: calculate_geometric_range

설명:
수학적인 등비수열의 규칙성과 특정 구간 합산 로직을 코드로 정밀하게 구현한 핵심 함수입니다. 이 함수는 첫째항부터 나아가는 현재 항의 값을 담을 변수(current_term = a)와 구간의 합을 누적할 변수(total = 0)를 초기화한 뒤, for i in range(1, m + 1): 반복문을 실행합니다. 1번째 항부터 사용자가 지정한 끝 항인 m번째 항까지 차례대로 루프를 돌면서, 내부에 위치한 조건문 if n <= i <= m:을 통해 현재 항 번호(i)가 사용자가 연산하기를 원하는 구간에 포함되는지 검사합니다. 조건이 참일 때만 total += current_term을 수행하여 값을 더하고, 조건 검사가 끝나면 current_term *= r 연산을 통해 다음 반복을 위한 등비수열의 다음 항 값을 준비합니다. 루프가 모두 종료되면 최종 누적된 값(total)을 호출부로 반환합니다. 이 함수 덕분에 공식으로 풀기 까다로운 등비수열의 중간 구간 합을 완벽하게 자동 계산할 수 있습니다

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

메인 루프 내부에서 사용자가 키보드로 입력한 메뉴 번호(k)가 1번, 2번, 3번, 4번 중 무엇인지 검사하여 대상을 분기하고, 범위 외의 숫자일 경우 안내 메시지를 띄우는 조건을 검사했습니다. (if-elif-else)

사용자가 수열 계산을 위해 입력한 구간 정보 중 시작 항 번호가 끝 항 번호보다 큰 오류 상황(if n > m)을 검사하여 차단했습니다.

구간 합 계산 함수 내부에서 수열이 전진할 때, 현재 항의 번호(i)가 사용자가 요구한 시작 항(n) 이상이자 끝 항(m) 이하의 범위에 속하는지(if n <= i <= m) 매 루프마다 검사했습니다.

기록 출력 함수에서 현재까지 누적된 기록의 개수(len(history_list) == 0)를 검사하여 저장된 이력이 없는 예외 상황을 구별했습니다.


### 반복문 사용 부분

사용자가 종료 번호인 4번을 입력하기 전까지 프로그램이 꺼지지 않고 계속해서 메뉴판을 띄우며 대기하도록 전체 실행 흐름을 while True 구문으로 무한 반복 처리했습니다.

등차수열과 등비수열을 계산할 때, 첫째항부터 사용자가 지정한 끝 항(m)까지 한 단계씩 항의 값을 갱신해 나가며 전진하는 과정을 for i in range(1, m + 1) 구문으로 반복 처리했습니다.

3번 메뉴를 통해 계산 기록을 조회할 때, 2차원 리스트 history에 쌓여있는 1차원 기록 상자(행 데이터)들을 처음부터 끝까지 하나씩 순서대로 꺼내오는 과정을 for info in history_list 구문으로 반복 처리했습니다.

### 조건문과 반복문이 함께 작동한 부분
수열 구간 합 계산 함수(calculate_arithmetic_range, calculate_geometric_range) 내부에서 두 제어구조가 가장 유기적이고 핵심적으로 결합하여 작동합니다. for 반복문은 수열의 규칙에 따라 1번째 항부터 마지막 m번째 항까지 멈추지 않고 데이터를 전진시키며 생성하는 역할을 수행합니다. 그리고 그 반복문의 품 안에 중첩되어 있는 if 조건문은 그렇게 매 단계마다 생성되는 현재 항 번호(i)를 감시하면서, 오직 사용자가 원했던 시작 구간 n과 끝 구간 m 사이에 들어오는 순간에만 필터링을 가해 누적 변수에 값을 더해주는 제어를 수행합니다. 즉, 반복문이 '데이터의 생성과 흐름'을 담당하고 조건문이 '데이터의 선택적 필터링'을 담당하여 원하는 구간의 합을 완벽히 도출해 냅니다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례

입력: 2번 메뉴(등비수열) 선택 ➡️ 첫째항(a)=3, 공비(r)=2, 시작항(n)=1, 끝항(m)=3 입력

예상 결과: 1번째 항(3), 2번째 항(6), 3번째 항(12)의 합이므로 3 + 6 + 12 = 21.0 출력 및 기록 저장

실제 결과: "첫째항이 3.0이고 공비가 2.0인 등비수열의 1번째 항부터 3번째 항까지의 합은 21.0입니다." 정상 출력 완료 및 3번 메뉴에서 이력 조회 확인 성공

### 테스트 2: 예외 또는 경계 사례

입력: 1번 메뉴(등차수열) 선택 ➡️ 첫째항(a)=2, 공차(d)=3 입력 후 시작항(n)=5, 끝항(m)=2 입력 (시작항이 끝항보다 큰 논리 오류 유도)

예상 결과: "오류: 시작 항 번호는 끝 항 번호보다 작거나 같아야 합니다!" 경고 메시지 출력 후 프로그램이 꺼지지 않고 메인 메뉴로 안전하게 복귀(continue)

실제 결과: 의도한 에러 메시지가 정상적으로 출력되었으며, 프로그램이 비정상 종료되거나 튕기지 않고 메인 메뉴판을 다시 안전하게 띄움

### 발견한 오류와 수정한 방법
오류 1 (NameError - 변수 미정의): 등비수열(k == 2) 구역을 처음 조립했을 때, 사용자에게 값을 입력받는 input() 코드보다 수열 계산 함수를 호출하는 result = calculate_geometric_range(...) 코드가 위쪽에 배치되어 파이썬이 a 변수를 인식하지 못하고 프로그램이 다운되는 오류가 발생했습니다. 파이썬의 하향식(위에서 아래로) 실행 특성을 고려하여 입력 코드를 함수 호출부 상단으로 올바르게 재배치하여 수정했습니다.

오류 2 (NameError - result 미정의): 계산 기록을 저장하는 history.append() 코드가 if-elif 조건문의 울타리 바깥(맨 아래)에 위치하여, 사용자가 프로그램을 켜자마자 계산을 안 하고 3번(기록보기)을 누르면 아직 생성되지 않은 result 변수를 참조하려다 프로그램이 터지는 결함이 있었습니다. 이를 해결하기 위해 history.append() 코드를 result가 확실하게 계산되어 살아있는 k == 1과 k == 2 내부 울타리 안쪽으로 각각 이동시켜 결함을 완벽히 해결했습니다.

## 8. Gemini 활용 내용 요약

프로그램의 뼈대가 되는 while True 메인 반복문 인터페이스의 들여쓰기(Indentation) 오류와 오타 자가 진단 및 디버깅 도움을 받았습니다.

수열의 수학적 메커니즘을 파이썬 코드로 변환할 때, for문과 if문을 결합하여 특정 구간만 깔끔하게 필터링하여 합산해 내는 효율적인 함수 구조 알고리즘을 제안받았습니다.

수행평가의 핵심 필수 조건인 '2차원 리스트 활용'을 만족하기 위해 계산 이력을 리스트 형태로 history.append()하여 다차원 배열 구조로 데이터를 누적하고, 이를 다시 for문 인덱싱으로 꺼내어 출력하는 정석적인 구현 방법을 전수받았습니다.

Gemini가 제안했지만 내가 수정하거나 사용하지 않은 부분:

Gemini는 처음에 수열 구간의 매개변수 이름을 알아보기 쉽게 start_n과 end_n으로 길게 작성할 것을 제안했으나, 수학 교과서에 나오는 정통 수열 기호 표현과 일치시키고 코드의 간결성을 높이기 위해 내가 직접 n과 m이라는 짧고 명확한 변수명으로 수정하여 채택했습니다.
또한 정보시간에 안배운 문법은 사용안했다
최종 코드에서 내가 직접 판단하거나 수정한 부분:

등비수열 계산 로직 함수(calculate_geometric_range) 내부에서 다음 항으로 진전할 때 빈칸으로 비워져 있던 공비 연산 기호 부분을, 등비수열의 수학적 원리(이전 항에 공비를 곱한다)에 착안하여 파이썬의 곱셈 축약 연산자인 *= r (current_term *= r) 형태로 스스로 추론하여 코드를 완성했습니다. 또한 각 메뉴의 가독성을 높이기 위해 대괄호([])와 이모지, 줄바꿈 기호를 조화롭게 배치하는 UI 디자인 요소를 직접 수정했습니다.

## 9. 한계와 개선 방안
현재 프로그램의 한계: 본 프로그램은 '등차수열'과 '등비수열' 두 가지 규칙을 가진 수열의 구간 합만 계산할 수 있도록 설계되어 있습니다. 이로 인해 현대 수학이나 컴퓨터 과학에서 다루는 피보나치수열, 계차수열, 혹은 사용자가 임의로 일반항을 정의하는 '그 외의 다양한 규칙을 가진 수열'에 대해서는 구간 합을 계산할 수 없다는 기능적 한계가 있습니다.

다시 개선한다면 추가하고 싶은 기능:이러한 한계를 극복하기 위해, 향후 프로그램 개발 시 사용자가 직접 화면에 수열의 일반항(예: $a_n = n^2 + 2n$)을 수식 형태로 입력할 수 있는 기능을 추가하고 싶습니다. 시그마 공식으로 계산할수 있도록.
